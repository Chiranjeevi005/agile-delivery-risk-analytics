# Stakeholder Map & Needs

## Stakeholder Hierarchy

### Primary Stakeholders
* **Product Manager**: Focuses on feature delivery and roadmap alignment.
* **Scrum Master**: Responsible for removing blockers and ensuring delivery health.
* **Engineering Team**: Executes technical tasks and manages individual capacity.

### Secondary Stakeholders
* **Business Leadership**: Depends on delivery timelines for strategic planning.
* **Customers**: End-users impacted by delay in feature availability.

## Stakeholder Matrix

| Stakeholder | Goal | Pain Point | Success Criteria |
| :--- | :--- | :--- | :--- |
| **Product Manager** | Predictable feature release | Unforeseen delays in roadmap | >90% Feature commitment accuracy |
| **Scrum Master** | Efficient sprint flow | Late-sprint "crisis" mode | Zero critical blockers by mid-sprint |
| **Engineering Team** | Sustainable work pace | Last-minute crunch due to overflow | Consistent velocity and low WIP |
| **Leadership** | Resource ROI transparency | Lack of visibility into delivery health | Reliable quarterly predictability |
| **Customers** | Timely feature access | Delayed value realization | Delivery on promised release dates |
