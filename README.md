# Agile Delivery Risk Analytics

Capability: Sprint Performance Analytics | Delivery Risk Modeling | Agile Workflow Optimization

---

### Overview

This project analyzes Agile sprint performance to identify delivery inefficiencies, quantify execution risk, and recommend data-driven improvements.

It simulates a real Business Analyst workflow by integrating:

- Data analysis (Excel + Power BI)
- Risk modeling
- Agile backlog creation (Jira simulation)
- Business documentation (BRD & FRD)
- Insight generation and recommendations

---

## Problem Statement

Sprint execution analysis revealed:

- Only ~25% of planned work is completed within sprint
- High spillover of unfinished work
- Unstable and declining sprint velocity
- Lack of data-driven planning

These issues lead to:

- Poor delivery predictability
- Increased execution risk
- Reduced stakeholder confidence

---

## Objective

To improve sprint delivery predictability by:

- Aligning sprint planning with actual capacity
- Reducing spillover
- Identifying delivery risk early
- Enabling data-driven decision-making

## Key Metrics

| Metric                 | Definition                       |
| ---------------------- | -------------------------------- |
| Commitment Reliability | Completed / Planned work         |
| Spillover Rate         | Unfinished / Planned work        |
| Velocity               | Work completed within sprint     |
| Risk Level             | Based on reliability & spillover |

---

## Key Insights

- Only **25% of planned work is delivered**
- Significant execution gap (12 planned vs 3 delivered)
- Presence of spillover indicates incomplete execution
- Velocity instability suggests capacity mismatch
- Overall delivery risk classified as **Critical (High Confidence)**

---

## Impact Analysis

- 75% of planned work remains incomplete
- Increasing backlog due to spillover
- Unpredictable delivery timelines
- High operational and execution risk

---

## Recommendations

- Reduce sprint commitment by ~60–70%
- Align planning with historical velocity
- Introduce mid-sprint progress tracking
- Improve workload distribution
- Implement real-time risk monitoring
